Place the DCC OpenAPI export as **`openapi.json`**.

It is referenced from **`config/projects.yaml`** (`openapi_relative` for project `dcc`) and used by:

- **`framework/contract_runner`** — discover / generate / run autogenerated GET cases
- AI context (`build_project_context_bundle`), when applicable

**Discovery-driven filters:** `discover_dcc` calls `GET /subject`, `/sample`, and `/file` with `page=1`, `per_page=50`, then merges up to **50** rows per entity type to build `filter_examples` (OpenAPI query param names → example strings from each row's `metadata`). `generate_cases_dcc` emits up to **20** additional filter cases per list path (`MAX_FILTER_EXTRA_CASES_PER_OP` in `dcc_generator.py`), each adding a single filter on top of default pagination. Optional `--strict-filter-data` on the contract (or perf) CLI sets `expect_non_empty_data` on those cases so the runner requires a non-empty `data` array.

Refresh this file when the public API contract changes.
